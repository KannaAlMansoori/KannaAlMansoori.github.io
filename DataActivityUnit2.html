Unit 2: Data Activity 2

Title: Frequency Analysis of COVID-19 Recovery Cases | Deadline: End of Unit 2 | Type: Formative | Status: ✓ Complete

Learning Outcomes from This Activity

  • Systematic understanding of the key mathematical and statistical concepts and techniques which underpin mechanisms in Data Science and AI.
  • Apply mathematical and statistical methods in these fields to help in the decision-making process.

Task Overview

Using the COVID-19 India Dataset (January 2020 - March 2020), perform frequency analysis activities to understand patterns in early pandemic reporting across Indian states.

Requirements:

  1. Create a binary variable called has_recovery that indicates whether any recoveries (cured cases) were reported
  2. Generate a frequency table to count how many daily reports showed recoveries versus no recoveries during this period using the table() command
  3. Calculate percentages to understand what proportion of daily state reports included recovery cases
  4. Record findings in e-portfolio with interpretation of results

Learning Outcomes Targeted:

  • Systematic understanding of key mathematical and statistical concepts and techniques which underpin mechanisms in Data Science and AI
  • Apply mathematical and statistical methods in these fields to help in the decision-making process

Methodology

Data Preparation

The analysis was conducted using R programming language. The COVID-19 India dataset covering January to March 2020 was loaded and prepared for frequency analysis.

R Code Implementation

Below is my complete R code for this activity:

# ============================================
# COVID-19 India Frequency Analysis
# Data Activity 2 - Unit 2
# ============================================

# Load required library
library(readr)

# Load the COVID-19 India dataset
Covid19_India_Jan_20_Mar_20_ <- read_csv("Numercial computer M3/Covid19 India (Jan 20 - Mar 20).csv")
View(Covid19_India_Jan_20_Mar_20_)

# Rename the dataset for easier handling
df <- Covid19_India_Jan_20_Mar_20_

# ============================================
# Step 1: Create a binary variable
# ============================================
# Create has_recovery variable: "Yes" if Cured > 0, "No" otherwise
df$has_recovery <- ifelse(df$Cured > 0, "Yes", "No")

# ============================================
# Step 2: Create a frequency table
# ============================================
# Generate frequency distribution
recovery_table <- table(df$has_recovery)
recovery_table

# ============================================
# Step 3: Calculate percentages
# ============================================
# Convert frequencies to percentages
recovery_percentage <- prop.table(recovery_table) * 100
recovery_percentage

Explanation of Key Functions

1. ifelse() Function

ifelse(df$Cured > 0, "Yes", "No")

This conditional function creates a binary classification: if the Cured count is greater than 0, it assigns "Yes"; otherwise, it assigns "No". This transforms a numeric variable into a categorical variable suitable for frequency analysis.

2. table() Function

table(df$has_recovery)

Generates a frequency table that counts how many observations fall into each category ("Yes" vs "No"). This is the foundation of categorical data analysis.

3. prop.table() Function

prop.table(recovery_table) * 100

Converts the frequency counts into proportions (0-1 scale), then multiplies by 100 to express as percentages. This allows us to understand the relative frequency of each category.

Analysis and Key Findings

1. Distribution Pattern

The frequency analysis reveals a highly skewed distribution, with the overwhelming majority (79.63%) of daily state reports showing no recovery cases during the January-March 2020 period. Only about one in five reports (20.37%) documented at least one recovery.

2. Early Pandemic Characteristics

This pattern reflects several aspects of the early pandemic phase:

  • Limited Outbreak Scale: The outbreak was just beginning to spread across India, with many states reporting zero or minimal cases
  • Recovery Time Lag: COVID-19 has an incubation period and recovery time, meaning patients infected in late January wouldn't recover until February or March
  • Case Concentration: Most cases were concentrated in a few hotspot states, while many states had no cases or recoveries to report
  • Data Collection Phase: Reporting systems were still being established and standardized across states

3. Statistical Significance

The 4:1 ratio (215:55) between non-recovery and recovery reports indicates that:

  • Recovery reporting was exceptional rather than routine during this period
  • The majority of observations (nearly 80%) represent days/states with zero recovery activity
  • This asymmetry suggests the disease was spreading faster than patients were recovering
  • The low recovery percentage aligns with the early exponential growth phase of the pandemic

4. Context in Pandemic Timeline

Understanding the January-March 2020 timeline:

  • January 2020: First cases detected in India (mostly travel-related)
  • February 2020: Gradual increase in cases, limited community transmission
  • March 2020: Acceleration of cases, first recoveries becoming more common
  • Result: Most of the dataset represents periods when active cases were accumulating faster than recoveries

5. Implications for Public Health Analysis

This frequency analysis demonstrates:

  • Data Quality Assessment: The binary variable helps identify gaps in recovery reporting and data completeness
  • Baseline Understanding: Establishes what "normal" looked like in early 2020, useful for comparing later pandemic phases
  • Resource Allocation: Shows that recovery case management wasn't the primary concern in early 2020; focus was on containment
  • Trend Identification: Provides a foundation for tracking how recovery patterns evolved as the pandemic progressed

Reflection

This activity reinforced the practical value of frequency analysis in understanding categorical patterns within datasets. While conceptually simpler than other statistical methods, frequency analysis proved remarkably powerful in revealing insights about the early pandemic period.

The most striking finding was the 79.63% to 20.37% split between reports without and with recoveries. This stark imbalance tells a clear story: during January-March 2020, COVID-19 in India was in its early exponential growth phase, with new infections far outpacing recoveries. The 4:1 ratio effectively quantifies what would otherwise be a vague observation about "limited recoveries."

Working with binary variables was particularly instructive. The ifelse() function's simplicity belies its power—with a single line of code, I transformed a numeric variable (Cured count) into a meaningful categorical classification. This demonstrated how data transformation is often the key to unlocking insights from raw data.

The exercise also highlighted the importance of context in statistical interpretation. Without understanding the pandemic timeline, the 79.63% figure might seem alarming. However, recognizing this as reflecting the natural lag between infection and recovery, combined with the early outbreak phase, provides appropriate context for the finding.

From a technical perspective, I appreciated how R's functions chain together naturally: ifelse() creates the variable, table() summarizes it, and prop.table() expresses it as percentages. This workflow represents a common pattern in data analysis—transform, summarize, interpret.

This activity has strengthened my understanding of how foundational statistical techniques underpin more complex data science applications. Frequency analysis may seem basic, but it's essential for understanding data distributions, identifying class imbalances, and making informed decisions in machine learning contexts.
