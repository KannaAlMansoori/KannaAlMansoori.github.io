Forum Post

Topic: LLM-Based Autonomous Agents

References: 4

Overview

The application I chose is the LLM-based autonomous agent, often called "agentic AI". This is a system that uses a large language model (LLM) as its "brain", but instead of just answering one question, it works towards a goal on its own. You give it a task, and it breaks it into smaller steps, uses tools such as web search, code or a database to carry them out, checks the result, and tries again if needed, without a person guiding every step (Wang et al., 2024). I picked this topic because it links deep learning with agent-based computing. The problems it raises, like autonomy, coordination and responsibility, are the same ones agent researchers have studied for years. The difference now is that the agent's "brain" is a deep learning model (Dignum and Dignum, 2026). It is also spreading quickly, with Gartner (2025) predicting that agents will make about 15% of routine work decisions by themselves within a few years, up from almost none in 2024.

How It Works

The deep learning part is the LLM itself, a large neural network trained on huge amounts of text, so it learns to predict language and reason with it. To turn this model into an agent, a few extra parts are added around it (Wang et al., 2024): a memory that keeps track of the conversation and past steps, a planning part that splits the goal into smaller tasks, and an action part that lets the agent use the tools. These parts work in a loop: the agent thinks about what to do, takes an action, looks at the result, and then thinks again, repeating this "think, act, observe" cycle until the job is finished. Larger systems use several agents together, each with its own role, much like how older multi-agent systems were designed.

Potential Impacts

The good side is clear. Agents can handle full tasks from start to finish, not just single steps, which saves time and gives more people access to skills they may not have. But this has an economic cost too, because if agents take over whole workflows, some jobs made up of those tasks may be reduced or replaced. The biggest risks, though, come from the same thing that makes them useful, which is their freedom to act. Because an agent acts instead of only giving advice, it can cause harm directly: Fang et al. (2024) showed that an agent built on GPT-4 could find and attack weak points in websites on its own, without being told about them first. Responsibility is another problem. When a long chain of steps ends badly, working out where the fault lies is not simple, since the user, the team that built the agent and the model all played a part. In a system with many agents, one mistake can also spread before anyone notices (Dignum and Dignum, 2026). Bias is a further issue, because the LLM is trained on internet text and picks up its unfair patterns, and since an agent acts, this bias can turn into a real decision, such as sorting job applications. Privacy is also a worry, because to be useful these agents are often given access to emails, files and calendars and remember past interactions, which puts a lot of private data in one place.

Overall I think the effect will be big, in both good and bad ways, and the answer is not only a technical one. As Dignum and Dignum (2026) argue, giving a system freedom without clear rules for control and responsibility is risky. So the focus should be on being open about how these agents work, keeping humans in control for important decisions, and having proper rules in place. Gartner (2025) also predicts that by the end of 2027, over four in ten of these projects will be dropped, because they cost too much, show no clear benefit, or cannot be controlled safely. So the worries are real in business, not only in theory.

Analysing an Agent-Based Application

What I did: I researched LLM-based autonomous agents and explained what they do, how they are built, and what their wider effects are likely to be. I supported the post with four sources, including a survey paper, a security study, an academic position paper, and industry forecasting.

Comparing approaches: Writing this post made me see how much of the classical agent architecture survives in these systems. The think–act–observe loop is a modern version of the sense–decide–act cycle, and the memory, planning and action components map onto parts that earlier agent designs also needed. The difference is where the reasoning comes from. In older architectures it was written by a developer as rules or as an explicit goal model; here it is learned from text and cannot be inspected directly. Multi-agent versions repeat the older multi-agent structure of specialised agents with separate roles, which is why the coordination problems look familiar too.

What I concluded: The property that makes these agents useful, their freedom to act, is also what makes them risky. Fang et al. (2024) demonstrated an agent finding and exploiting website vulnerabilities without being directed to them, which shows the difference between a system that advises and a system that acts. Responsibility becomes hard to assign across a long chain of steps involving the user, the developers and the model. My conclusion was that transparency, human control over important decisions, and clear rules matter as much as capability, and Gartner's (2025) forecast that over 40% of agentic projects will be cancelled by the end of 2027 suggests organisations are already meeting these limits in practice.

Reflection

The most useful part of this activity was realising that agentic AI is not a break from what we studied earlier in the module. It is the same architecture with a different reasoning engine. That made the older material feel relevant rather than historical.

Finding sources was harder than writing. This is a fast-moving area, so much of what is available is marketing rather than research. I looked for a peer-reviewed survey for the technical description, an experimental paper for the security claim, and an industry forecast for adoption, so that each type of claim was supported by an appropriate kind of source.

Coming from statistics, the point that stayed with me is the shift from advice to action. A wrong prediction can be checked before anyone uses it. An agent that acts on a wrong prediction has already done something. That changes what oversight has to mean, and it is a question I want to bring into the Unit 12 seminar.
