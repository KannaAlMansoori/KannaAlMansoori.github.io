Unit 6 Seminar Preparation

Working Together: KQML Dialogues

Type: Seminar Preparation | Status: ✓ Complete

What I Already Know (CD2)

1. Agents Can Refuse (Autonomy)

From CD2: Agents don't have to do what you ask. They can refuse, delay, or negotiate. That's the key difference from method calls.

2. Messages Have Intention (Performatives)

From CD2: When I "ask" something, I'm requesting. When I "tell," I'm informing. Each has meaning beyond just words.

3. Ontology Problem (Biggest Challenge)

From CD2: If agents don't share the same meaning of words, they misunderstand silently. This was identified as the biggest limitation.

4. Why FIPA-ACL Was Created

From CD2: KQML lacked formal semantics. FIPA-ACL fixed this by being clearer about what each message means.

What to Focus On in the Seminar

1. Performatives in Real Dialogues

Focus: Identify ask, tell, reply, achieve, subscribe in examples

Why: The seminar will show real conversations. Can I spot each performative?

2. Why Agents Refuse/Negotiate

Focus: When would an agent say "no"? Why?

Why: This is agent autonomy in action. Different from method calls.

3. Semantic Misalignment Examples

Focus: Watch for cases where agents misunderstand each other

Why: From CD2, this is the hardest problem to solve.

4. When to Use KQML vs Method Calls

Focus: Why would you pick KQML for a problem?

Why: Understand the practical trade-offs.

What to Enhance Understanding On

Quick Example: See the Difference

Java Method Call

result = payrollAgent.calculateSalary(empID);
// Agent must execute immediately, no choice

KQML Message

Employee → ask → Payroll Agent: "What is my salary?"
Payroll → reply → Employee: "Your salary is $50,000"
// Agent chose to answer. Could have refused.

Key difference: KQML = request with choice. Java = command with no choice.
