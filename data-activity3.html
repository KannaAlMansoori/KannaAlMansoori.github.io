Task Overview

This activity extends the frequency analysis from Data Activity 2 by creating more complex categorical variables and analyzing geographic patterns in the COVID-19 India Dataset (January 2020 - March 2020).

Requirements:

  1. Create a binary variable called has_deaths that indicates whether any deaths were reported:
    • Value = 1 (or TRUE) if Deaths > 0
    • Value = 0 (or FALSE) if Deaths = 0
  2. Create a frequency table using the table() command to analyze death reporting patterns
  3. Convert to a factor variable with appropriate labels ("No Deaths", "Deaths Reported")
  4. Create a categorical variable called case_level based on total confirmed cases (Indian + Foreign nationals):
    • No Cases = 0 cases
    • Low Cases = 1-5 cases
    • Medium Cases = 6-15 cases
    • High Cases = 16+ cases
  5. Create a frequency table for the State/Union Territory variable to see which states had the most frequent reporting
  6. Identify the top 10 states with the highest number of daily reports in the dataset

Learning Outcomes Targeted:

  • Systematic understanding of the key mathematical and statistical concepts and techniques which underpin mechanisms in Data Science and AI.
  • Apply mathematical and statistical methods in these fields to help in the decision-making process.

Methodology

Data Preparation

Building upon the previous activities, this analysis creates multiple categorical variables to classify and understand the severity and geographic distribution of COVID-19 cases during the early pandemic period.

R Code Implementation

Below is my complete R code for this activity:

# ============================================
# COVID-19 India Categorical Variables Analysis
# Data Activity 3 - Unit 2
# ============================================

# Load required library
library(readr)

# Load the COVID-19 India dataset
Covid19_India_Jan_20_Mar_20_ <- read_csv("Numercial computer M3/Covid19 India (Jan 20 - Mar 20).csv")
View(Covid19_India_Jan_20_Mar_20_)

# Rename the dataset for easier handling
df <- Covid19_India_Jan_20_Mar_20_

# ============================================
# Step 1: Binary variable - has_deaths
# ============================================
# Create binary indicator: 1 if Deaths > 0, 0 otherwise
df$has_deaths <- ifelse(df$Deaths > 0, 1, 0)

# ============================================
# Step 2: Frequency table for deaths
# ============================================
# Generate frequency distribution
table(df$has_deaths)

# ============================================
# Step 3: Convert to factor with labels
# ============================================
# Convert binary variable to labeled factor
df$has_deaths <- factor(
  df$has_deaths,
  levels = c(0, 1),
  labels = c("No Deaths", "Deaths Reported")
)
table(df$has_deaths)

# ============================================
# Step 4: Calculate total confirmed cases
# ============================================
# Sum Indian and Foreign national cases
df$total_cases <- df$ConfirmedIndianNational +
  df$ConfirmedForeignNational

# ============================================
# Step 5: Create categorical variable - case_level
# ============================================
# Classify cases into severity levels
df$case_level <- cut(
  df$total_cases,
  breaks = c(-1, 0, 5, 15, Inf),
  labels = c("No Cases", "Low Cases", "Medium Cases", "High Cases")
)
table(df$case_level)

# ============================================
# Step 6: Frequency table by State/Union Territory
# ============================================
# Count reports per state
state_freq <- table(df$`State/UnionTerritory`)
state_freq

# ============================================
# Step 7: Top 10 states with most daily reports
# ============================================
# Sort and extract top 10
top_10_states <- sort(state_freq, decreasing = TRUE)[1:10]
top_10_states

Explanation of Key Functions

1. ifelse() for Binary Classification

ifelse(df$Deaths > 0, 1, 0)

Creates a binary indicator where 1 represents presence of deaths and 0 represents absence. This is the foundation for binary classification in data science.

2. factor() for Labeled Categories

factor(df$has_deaths, levels = c(0, 1), labels = c("No Deaths", "Deaths Reported"))

Converts numeric values to meaningful categorical labels, making the data more interpretable and suitable for categorical analysis.

3. cut() for Multi-level Classification

cut(df$total_cases, breaks = c(-1, 0, 5, 15, Inf), labels = c(...))

Divides continuous numeric data into discrete categories based on specified breakpoints. The breaks define the boundaries: -1 to 0 (No Cases), 0 to 5 (Low), 5 to 15 (Medium), 15+ (High).

4. sort() with Indexing

sort(state_freq, decreasing = TRUE)[1:10]

Sorts the frequency table in descending order and extracts the top 10 entries, useful for identifying the most active reporting states.

Analysis and Key Findings

1. Death Reporting Patterns

What I Found: Only 9 out of 270 daily reports (3.33%) documented any deaths. The vast majority—261 reports (96.67%)—showed zero deaths.

What This Means:

This extremely low death rate makes sense for January-March 2020. COVID-19 had just arrived in India, so most cases were recent infections. Patients hadn't reached critical stages yet, and healthcare systems were able to respond quickly to the limited number of cases. This data shows we're looking at the very beginning of the pandemic, not the widespread outbreak that came later.

2. Case Severity Distribution

What I Found: Most daily reports (52.96%) showed low case numbers (1-5 cases). Medium cases (6-15) made up 21.11%, while high cases (16+) were rare at only 6.67%. Nearly 20% of reports had no cases at all.

What This Means:

The data clearly shows the pandemic was in its early containment phase. Over 72% of reports had 5 or fewer cases—this wasn't widespread community transmission yet. High-case days were exceptions, not the norm. This classification system effectively identifies which states needed immediate resources and which were still in the monitoring phase.

3. Geographic Patterns: Kerala's Story

What I Found: Kerala dominated with 52 daily reports—more than double any other state. Delhi and Telengana tied for second place with 20 reports each. The top 10 states accounted for about 77% of all reports.

Why Kerala Led:

Kerala was India's COVID-19 entry point. The state reported India's first cases from travelers returning from China, and it has strong international connections, especially with the Middle East. But Kerala's high numbers also reflect something positive: they had excellent public health infrastructure and were proactive about testing and reporting. While other states might have had undetected cases, Kerala was actively looking for them.

The concentration in urban centers like Delhi, Mumbai (Maharashtra), and Bangalore (Karnataka) makes sense—these are major international hubs where the virus would naturally arrive first through travelers.

4. What I Learned About Data Analysis

This activity showed me how powerful simple categorization can be:

  • Creating the has_deaths binary variable simplified a complex question into yes/no, making patterns immediately visible
  • The case_level categories transformed raw numbers into severity levels that anyone can understand
  • Converting numbers to labeled factors ("No Deaths" vs 0) made my analysis accessible to people without statistical backgrounds
  • Simple frequency tables revealed geographic patterns that would be hard to spot in raw data

5. Real-World Applications

Understanding these patterns helps with practical decisions:

  • Resource Planning: States with high case counts need immediate medical supplies and personnel
  • Risk Communication: The 96.67% "no deaths" figure helps explain that early 2020 was different from later pandemic waves
  • Geographic Strategy: Focusing on Kerala and major urban centers made sense given the data
  • Baseline Comparison: These early numbers provide a benchmark to measure how the pandemic evolved

This technique makes raw data much easier to interpret and use for decision-making.

Working with the binary has_deaths variable and converting it to labeled factors showed me the importance of clear data presentation. Using "No Deaths" and "Deaths Reported" instead of 0 and 1 makes results immediately understandable, especially when sharing findings with non-technical audiences.

The geographic analysis was eye-opening—seeing Kerala with 52 reports (nearly 20% of all data) clearly demonstrated how simple frequency tables can reveal important patterns. The finding that 96.67% of reports had no deaths and 52.96% showed low case counts perfectly captured the early pandemic phase in India. These statistical methods proved that straightforward analysis, when applied correctly, can provide powerful insights for public health decision-making.
