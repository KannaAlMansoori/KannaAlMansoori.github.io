Task Overview

This activity extends the frequency analysis from Data Activity 2 by creating more complex categorical variables and analyzing geographic patterns in the COVID-19 India Dataset (January 2020 - March 2020).

Requirements:

  1. Create a binary variable called has_deaths that indicates whether any deaths were reported:
    • Value = 1 (or TRUE) if Deaths > 0
    • Value = 0 (or FALSE) if Deaths = 0
  2. Create a frequency table using the table() command to analyze death reporting patterns
  3. Convert to a factor variable with appropriate labels ("No Deaths", "Deaths Reported")
  4. Create a categorical variable called case_level based on total confirmed cases (Indian + Foreign nationals):
    • No Cases = 0 cases
    • Low Cases = 1-5 cases
    • Medium Cases = 6-15 cases
    • High Cases = 16+ cases
  5. Create a frequency table for the State/Union Territory variable to see which states had the most frequent reporting
  6. Identify the top 10 states with the highest number of daily reports in the dataset

Learning Outcomes Targeted:

  • Systematic understanding of the key mathematical and statistical concepts and techniques which underpin mechanisms in Data Science and AI.
  • Apply mathematical and statistical methods in these fields to help in the decision-making process.

Methodology

Data Preparation

Building upon the previous activities, this analysis creates multiple categorical variables to classify and understand the severity and geographic distribution of COVID-19 cases during the early pandemic period.

R Code Implementation

Below is my complete R code for this activity:

# ============================================
# COVID-19 India Categorical Variables Analysis
# Data Activity 3 - Unit 2
# ============================================

# Load required library
library(readr)

# Load the COVID-19 India dataset
Covid19_India_Jan_20_Mar_20_ <- read_csv("Numercial computer M3/Covid19 India (Jan 20 - Mar 20).csv")
View(Covid19_India_Jan_20_Mar_20_)

# Rename the dataset for easier handling
df <- Covid19_India_Jan_20_Mar_20_

# ============================================
# Step 1: Binary variable - has_deaths
# ============================================
# Create binary indicator: 1 if Deaths > 0, 0 otherwise
df$has_deaths <- ifelse(df$Deaths > 0, 1, 0)

# ============================================
# Step 2: Frequency table for deaths
# ============================================
# Generate frequency distribution
table(df$has_deaths)

# ============================================
# Step 3: Convert to factor with labels
# ============================================
# Convert binary variable to labeled factor
df$has_deaths <- factor(
  df$has_deaths,
  levels = c(0, 1),
  labels = c("No Deaths", "Deaths Reported")
)
table(df$has_deaths)

# ============================================
# Step 4: Calculate total confirmed cases
# ============================================
# Sum Indian and Foreign national cases
df$total_cases <- df$ConfirmedIndianNational +
  df$ConfirmedForeignNational

# ============================================
# Step 5: Create categorical variable - case_level
# ============================================
# Classify cases into severity levels
df$case_level <- cut(
  df$total_cases,
  breaks = c(-1, 0, 5, 15, Inf),
  labels = c("No Cases", "Low Cases", "Medium Cases", "High Cases")
)
table(df$case_level)

# ============================================
# Step 6: Frequency table by State/Union Territory
# ============================================
# Count reports per state
state_freq <- table(df$`State/UnionTerritory`)
state_freq

# ============================================
# Step 7: Top 10 states with most daily reports
# ============================================
# Sort and extract top 10
top_10_states <- sort(state_freq, decreasing = TRUE)[1:10]
top_10_states

Explanation of Key Functions

1. ifelse() for Binary Classification

ifelse(df$Deaths > 0, 1, 0)

Creates a binary indicator where 1 represents presence of deaths and 0 represents absence. This is the foundation for binary classification in data science.

2. factor() for Labeled Categories

factor(df$has_deaths, levels = c(0, 1), labels = c("No Deaths", "Deaths Reported"))

Converts numeric values to meaningful categorical labels, making the data more interpretable and suitable for categorical analysis.

3. cut() for Multi-level Classification

cut(df$total_cases, breaks = c(-1, 0, 5, 15, Inf), labels = c(...))

Divides continuous numeric data into discrete categories based on specified breakpoints. The breaks define the boundaries: -1 to 0 (No Cases), 0 to 5 (Low), 5 to 15 (Medium), 15+ (High).

4. sort() with Indexing

sort(state_freq, decreasing = TRUE)[1:10]

Sorts the frequency table in descending order and extracts the top 10 entries, useful for identifying the most active reporting states.

Analysis and Key Findings

1. Death Reporting Patterns

Key Finding: Only 3.33% (9 out of 270) of daily reports documented deaths, while 96.67% (261 reports) showed no deaths.

Interpretation:

  • Early Pandemic Phase: This extremely low death rate reflects the January-March 2020 timeframe when the outbreak was just beginning in India
  • Limited Fatalities: With most cases being recent infections, few patients had reached critical stages by this period
  • Healthcare Response: Early detection and treatment may have contributed to low mortality rates
  • Data Context: Death reporting may have been inconsistent or incomplete in the early stages of the pandemic

2. Case Severity Distribution

Distribution Pattern:

  • Low Cases (1-5): 52.96% - The largest category, indicating most reporting days involved minimal cases
  • Medium Cases (6-15): 21.11% - Moderate case loads in about one-fifth of reports
  • No Cases: 19.26% - Nearly 20% of reports showed zero cases
  • High Cases (16+): 6.67% - Only a small fraction had significant case numbers

Key Insights:

  • The distribution is heavily skewed toward lower case counts, with 72.22% of reports showing 5 or fewer cases
  • High-case scenarios (16+) were rare exceptions rather than the norm during this period
  • This pattern indicates the pandemic was in its early containment phase rather than widespread community transmission
  • The classification system effectively identifies severity levels for resource allocation and response planning

3. Geographic Distribution Analysis

Top Finding: Kerala led with 52 daily reports - more than double any other state.

Geographic Patterns:

  • Kerala's Dominance: With 52 reports (19.26% of all reports), Kerala had by far the most frequent reporting, reflecting its role as an early entry point for COVID-19 into India through international travel
  • Tier 2 States: Delhi and Telengana (20 reports each) formed the second tier, representing major urban centers with international connections
  • Distributed Spread: The top 10 states/UTs account for approximately 77% of all reports, showing concentration in specific regions while most states had minimal reporting
  • Urban Concentration: Most top-reporting states are major urban centers or have significant international airports

Why Kerala Led:

  • First state to report COVID-19 cases in India (from travelers returning from Wuhan)
  • High international connectivity, especially with the Middle East
  • Strong public health infrastructure and early detection systems
  • Proactive testing and reporting compared to other states

4. Statistical Methodology Insights

This activity demonstrated several important data science concepts:

  • Binary Classification: The has_deaths variable shows how continuous or count data can be simplified into yes/no categories for specific analytical purposes
  • Multi-level Categorization: The case_level variable demonstrates how cut() function creates meaningful severity categories from numeric data
  • Factor Variables: Converting to labeled factors makes data more interpretable and prepares it for categorical analysis
  • Frequency Analysis: Simple counting and sorting revealed clear geographic patterns that would be harder to see in raw data
  • Data Transformation: Creating derived variables (total_cases) from existing columns enabled more sophisticated classification

5. Implications for Decision-Making

These analyses support evidence-based decisions:

  • Resource Allocation: The case_level classification helps identify which states need more intensive support
  • Risk Assessment: Understanding that 96.67% of reports had no deaths indicates low immediate mortality risk in early 2020
  • Geographic Focus: Kerala's dominance suggests the need for concentrated efforts in high-reporting states
  • Trend Monitoring: Baseline categories enable tracking of how severity patterns evolve over time
  • Public Health Strategy: The concentration of cases in urban centers informs containment strategies

Reflection

This activity significantly expanded my understanding of categorical variable creation and manipulation in R. While Data Activity 2 introduced binary classification, this exercise demonstrated how to create more complex multi-level categories and conduct geographic analysis.

The most impactful learning came from working with the cut() function to create the case_level variable. Understanding how to define meaningful breakpoints (-1, 0, 5, 15, Inf) and assign appropriate labels transformed continuous case counts into actionable categories. This skill is fundamental in data science—converting raw numbers into classifications that support decision-making.

Converting the binary has_deaths variable into a labeled factor was particularly instructive. The transformation from numeric 0/1 to meaningful "No Deaths"/"Deaths Reported" labels demonstrates how proper data formatting improves interpretability. This is especially important when presenting findings to non-technical stakeholders who need to understand results quickly.

The geographic analysis revealed Kerala's dominant position with 52 reports—nearly 20% of all observations. This finding makes sense given Kerala's role as India's early COVID-19 entry point, but seeing it quantified so clearly through simple frequency analysis was powerful. It demonstrated how straightforward statistical methods can uncover significant patterns when applied thoughtfully.

The 96.67% "No Deaths" finding was striking but requires careful contextualization. Without understanding the January-March 2020 timeframe, this figure might be misinterpreted. This reinforced the importance of temporal context in data analysis—statistics alone don't tell the full story.

From a technical perspective, I appreciated how R's functions build upon each other. The workflow—calculate total_cases, use cut() to categorize, then table() to summarize—represents a logical progression that's applicable to many data science problems. This modular approach to data transformation is something I can apply across different domains.

The case severity distribution (52.96% Low Cases, 21.11% Medium, 6.67% High) perfectly illustrated the early pandemic phase. Most days saw minimal cases, with high-case days being rare exceptions. This pattern would change dramatically in later pandemic phases, making this baseline analysis valuable for understanding how the situation evolved.

This activity reinforced that effective data analysis isn't just about applying functions—it's about choosing appropriate categorizations, understanding the context behind the numbers, and presenting findings in ways that support decision-making. These skills are essential for any data science role, particularly in public health where accurate interpretation can influence policy decisions affecting millions of people.
