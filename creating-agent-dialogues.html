Creating Agent Dialogues

Alice and Bob: KQML and KIF Communication

Activity: Agent Dialogue Creation | Status: ✓ Complete

The Task

Create a KQML dialogue between Alice (stock procurement agent) and Bob (warehouse stock control agent).

  • Alice asks Bob about available stock of 50 inch televisions
  • Alice queries the number of HDMI slots on the televisions

Alice and Bob Dialogue

Step 1: Alice Asks About Stock

Alice: "How many 50 inch televisions do you have in stock?"

(ask
  :sender alice
  :receiver bob
  :content (quantity-in-stock ?qty Television-50inch)
  :reply-with stock-query-1
)

Step 2: Bob Replies with Stock Level

Bob: "I have 24 of those 50 inch TVs in stock."

(reply
  :sender bob
  :receiver alice
  :in-reply-to stock-query-1
  :content (quantity-in-stock 24 Television-50inch)
)

Step 3: Alice Asks About HDMI Specifications

Alice: "How many HDMI slots do those TVs have?"

(ask
  :sender alice
  :receiver bob
  :content (hdmi-slots ?slots Television-50inch)
  :reply-with hdmi-query-1
)

Step 4: Bob Provides HDMI Information

Bob: "The 50 inch TVs have 3 HDMI slots each."

(reply
  :sender bob
  :receiver alice
  :in-reply-to hdmi-query-1
  :content (hdmi-slots 3 Television-50inch)
)

How This Works

KQML (How They Communicate)

KQML provides the structure:

  • ask: Alice requests information politely (Bob can choose to answer)
  • reply: Bob responds with the answer
  • :reply-with and :in-reply-to: Tags track which answers match which questions

KIF (What They Communicate)

KIF provides the meaning:

  • ?qty and ?slots: Variables representing unknowns Alice wants to discover
  • Television-50inch: Precise name so both agents understand exactly what product is being discussed
  • Removes ambiguity - both agents understand the same thing

Why This Matters

This dialogue shows:

  • Agent Autonomy: Bob could have refused or asked questions, but he chose to cooperate
  • Clear Communication: KQML + KIF ensures no misunderstandings
  • Modularity: Alice and Bob are independent systems that work together

Learning Outcomes

1. Motivations for Agent-Based Computing

This dialogue shows why agents are useful:

  • Independent agents can be built and updated separately
  • Agents maintain autonomy - they decide how to respond
  • Multiple agents working together solves complex problems
  • Scales from 2 agents to thousands

2. Agent Models and AI Grounding

Both types of agents shown here:

  • Bob (Reactive): Asked a question, responds with answer - simple stimulus-response
  • Alice (Deliberative): Has a goal (get TV info) and decides how to achieve it
  • Standards (KQML + KIF): Grounded in AI research on communication and knowledge representation

What I Learned From This

From my CD2 discussion, I knew about KQML performatives. Creating this dialogue made it real:

  • I understood that agents can refuse - now I see how ask/reply actually works
  • I knew about the ontology problem - KIF fixes it by being precise about what we're talking about
  • I studied agent autonomy - Bob choosing to answer shows this in action
  • I read about distributed systems - Alice and Bob solving a problem together demonstrates this
