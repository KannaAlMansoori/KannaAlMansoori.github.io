Creating Agent Dialogues

Alice and Bob: KQML and KIF Communication

Activity: Formative | Type: Agent Dialogue Creation | Status: ✓ Complete

Activity Overview

Task

Create a KQML dialogue between two agents:

  • Alice: Stock procurement agent (buyer)
  • Bob: Warehouse stock control agent (supplier)

Dialogue Requirements

  • Alice asks Bob about available stock of 50 inch televisions
  • Alice queries the number of HDMI slots on the televisions
  • Use KQML performatives for communication
  • Use KIF (Knowledge Interchange Format) for knowledge representation

Learning Connection

This activity brings together concepts from my Collaborative Discussion 2 where I explored how agents communicate using structured messages with clear intentions. Now I'm creating an actual dialogue to demonstrate this understanding.

Alice and Bob Dialogue

Scenario

Alice (procurement agent) needs to check warehouse stock levels for 50 inch televisions. She wants to know:

  • How many 50 inch TVs are in stock?
  • How many HDMI slots do they have?

KQML Dialogue: Stock Enquiry

Message 1: Alice Asks About Stock Quantity

Performative: ask

Sender: Alice (procurement agent)

Receiver: Bob (warehouse agent)

(ask
  :sender alice
  :receiver bob
  :content (quantity-in-stock ?qty Television-50inch)
  :reply-with stock-query-1
)

Meaning: Alice is asking Bob: "What is the quantity of 50 inch televisions in stock?"

KIF Knowledge Representation: The content uses KIF to express what Alice wants to know - a variable ?qty that represents the quantity of Television-50inch items.

Message 2: Bob Replies with Stock Quantity

Performative: reply

Sender: Bob (warehouse agent)

Receiver: Alice (procurement agent)

(reply
  :sender bob
  :receiver alice
  :in-reply-to stock-query-1
  :content (quantity-in-stock 24 Television-50inch)
)

Meaning: Bob is replying: "There are 24 units of 50 inch televisions in stock."

KIF Knowledge Representation: Bob binds the variable ?qty to the actual value 24, providing the concrete answer to Alice's question.

Message 3: Alice Asks About HDMI Slots

Performative: ask

Sender: Alice (procurement agent)

Receiver: Bob (warehouse agent)

(ask
  :sender alice
  :receiver bob
  :content (hdmi-slots ?slots Television-50inch)
  :reply-with hdmi-query-1
)

Meaning: Alice is asking Bob: "How many HDMI slots do the 50 inch televisions have?"

KIF Knowledge Representation: The variable ?slots represents the number of HDMI slots that Alice wants to know about.

Message 4: Bob Replies with HDMI Information

Performative: reply

Sender: Bob (warehouse agent)

Receiver: Alice (procurement agent)

(reply
  :sender bob
  :receiver alice
  :in-reply-to hdmi-query-1
  :content (hdmi-slots 3 Television-50inch)
)

Meaning: Bob is replying: "The 50 inch televisions have 3 HDMI slots."

KIF Knowledge Representation: Bob binds the variable ?slots to 3, providing the specific information about HDMI connectivity.

Analysis: How This Dialogue Works

Why KQML Performatives?

From my CD2 discussion, I learned that performatives express intention. In this dialogue:

  • ask: Alice requests information without commanding. Bob can choose to answer or refuse.
  • reply: Bob provides information in response. This shows agent autonomy—Bob decides what information to share.

Why KIF for Knowledge?

KIF allows precise representation of what Alice and Bob know:

  • Variables (? notation): ?qty and ?slots represent unknown values Alice wants to discover
  • Predicates: quantity-in-stock and hdmi-slots are facts about the televisions
  • Clear Semantics: Unlike raw KQML, KIF removes ambiguity about what "50 inch television" means

Connection to CD2: This addresses the ontology problem I discussed. KIF provides formal semantics so both agents understand exactly what they're talking about.

Agent Autonomy in Action

Notice that Bob could have:

  • Refused to answer (if stock information is confidential)
  • Asked for clarification (if "50 inch television" was ambiguous)
  • Negotiated (e.g., "I'll tell you the stock level if you agree to a minimum order")

Instead, Bob chose to cooperate and provide honest answers. This demonstrates agent autonomy—they're not blindly executing commands.

Practical Application

This dialogue could represent a real warehouse management system where:

  • Alice is a procurement software module
  • Bob is an inventory management system
  • They communicate through KQML to coordinate purchasing decisions
  • The knowledge is formally represented in KIF so there's no ambiguity

Learning Outcomes Demonstrated

Outcome 1: Motivations for Agent-Based Computing

This dialogue shows why agents are useful:

  • Modularity: Alice and Bob are independent agents that can be updated separately
  • Autonomy: Each agent makes decisions based on its own logic
  • Scalability: This pattern works for 2 agents or 2000 agents
  • Flexibility: Agents can negotiate, refuse, or modify requests

Outcome 2: Main Agent Models and AI Grounding

This dialogue demonstrates:

  • Reactive Agents: Bob responds to Alice's queries (stimulus-response)
  • Deliberative Agents: Alice decides what to ask based on her procurement goals
  • Communication Standards: KQML is grounded in AI research on agent communication (speech act theory)
  • Knowledge Representation: KIF formalizes knowledge in a way AI systems can understand

What This Activity Enhanced

From Theory to Practice

In my seminar preparation, I studied KQML examples. This activity forces me to create one, which deepens understanding of:

  • How to structure messages with sender, receiver, and content
  • How to use reply-with tags to track conversations
  • How variables in KIF represent unknowns that agents exchange

Real-World Thinking

Creating this dialogue made me think about:

  • What if Bob's warehouse was out of stock? (Agent autonomy in refusal)
  • What if Alice misunderstood what "50 inch" means? (Ontology clarity with KIF)
  • What if Bob wanted to negotiate terms? (Multi-turn dialogue complexity)

Technical Appreciation

I now understand why KQML and KIF exist:

  • KQML's structure (sender, receiver, performative) enables reliable agent communication
  • KIF's formal semantics prevent misunderstandings that could cost money in real systems
  • The combination of KQML (how to communicate) + KIF (what to communicate) is powerful
